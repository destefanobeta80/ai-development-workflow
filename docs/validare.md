# Fase 5: Validare

## Scopo

Garantire che il codice sia corretto, solido e difendibile.

---

## Descrizione

Questa fase include:
- test
- debugging
- review AI
- review umana
- miglioramento

È iterativa (loop).

Errori tipici:
- controllare solo “funziona?”
- ignorare edge case
- non rileggere il codice

---

## Prompt

Agisci come reviewer tecnico.

Valuta il codice su:
- correttezza
- allineamento al requisito
- qualità
- rischi
- edge case

Per ogni punto:
- problemi trovati
- severità
- suggerimenti

Alla fine:
- approvabile / non approvabile

---

## Condizione di uscita

Il codice è:
- corretto
- pulito
- coerente con il requisito

Non ci sono problemi evidenti non risolti.

Se trovi bug o dubbi, continua il loop.
