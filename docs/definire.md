# Fase 1: Definire

## Scopo

Stabilire cosa deve essere fatto, in modo chiaro e misurabile.

---

## Descrizione

Qui trasformi il problema in un obiettivo concreto.

Devi definire:
- cosa vuoi ottenere
- cosa NON vuoi fare
- quando consideri il lavoro finito

Errori tipici:
- obiettivi vaghi
- scope troppo ampio
- mancanza di criteri di successo

---

## Prompt

```text
Agisci come tech lead.

Partendo dal problema, definisci il risultato corretto.

Restituisci:
- Obiettivo
- Non obiettivi
- Acceptance criteria
- Vincoli tecnici
- Rischi principali
- Definition of done

Se qualcosa è ambiguo, evidenzialo.
Se è troppo grande, riduci lo scope.
```

## Condizione di uscita

Esiste una definizione chiara di:

cosa deve essere fatto
cosa non deve essere fatto
quando il task è completo

Un’altra persona dovrebbe poter capire cosa fare senza chiedere.
